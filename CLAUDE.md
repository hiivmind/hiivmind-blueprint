## Plugin Overview

**hiivmind-blueprint-author** provides journey-oriented tools for building, assessing, enhancing, extracting, maintaining, and visualizing Claude Code skills as deterministic YAML workflow patterns. This plugin handles the full skill lifecycle through 7 journey-oriented skills with a unified gateway.

## Architecture: Journey Model

This plugin uses a journey-oriented architecture where each skill represents a distinct phase of the skill authoring lifecycle. Users enter through the **gateway** skill, which routes to the appropriate journey skill based on intent.

### The 7 Journey Skills

| Skill | Journey Phase | Purpose |
|-------|--------------|---------|
| **bp-gateway** | Entry point | Routes user intent to the correct journey skill |
| **bp-build** | Creation | Build new skills from scratch with workflow patterns |
| **bp-assess** | Evaluation | Assess existing skills for quality, structure, and conversion readiness |
| **bp-enhance** | Improvement | Enhance existing skills with better patterns, error handling, and structure |
| **bp-extract** | Extraction | Extract workflow patterns from prose-based skills |
| **bp-maintain** | Maintenance | Maintain skills over time — validate, update, and repair |
| **bp-visualize** | Visualization | Generate Mermaid diagrams from workflow definitions |

### Archived Skills

The `_archived/` directory contains old skills from the pre-journey architecture (bp-skill-create, bp-skill-analyze, bp-plugin-discover, etc.). These are retained for reference but are no longer active.

## Skills

| Skill | Purpose | Invocation |
|-------|---------|------------|
| **bp-gateway** | Route intent to journey skill | `/hiivmind-blueprint-author [request]` |
| **bp-build** | Build new skills from scratch | `/hiivmind-blueprint-author build [description]` |
| **bp-assess** | Assess skill quality and structure | `/hiivmind-blueprint-author assess [path]` |
| **bp-enhance** | Enhance existing skills | `/hiivmind-blueprint-author enhance [path]` |
| **bp-extract** | Extract workflows from prose skills | `/hiivmind-blueprint-author extract [path]` |
| **bp-maintain** | Validate and maintain skills | `/hiivmind-blueprint-author maintain [path]` |
| **bp-visualize** | Generate Mermaid diagrams | `/hiivmind-blueprint-author visualize [path]` |

## Version Management

The external `hiivmind-blueprint-lib` version is centralized in `.hiivmind/blueprint/config.yaml`.
Skills reference the version dynamically via `{computed.lib_version}` at runtime.
